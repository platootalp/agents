# GitHub Stars 索引示例

本文档提供不同场景下的索引文档示例。

## 示例 1：极简风格索引

适用场景：收藏数量较少（<50个），只需要基本信息

```markdown

| Name | Description | Link |
|------|-------------|------|
| langchain | Building LLM applications | [GitHub](https://github.com/langchain/langchain) |
| llama.cpp | LLM inference in C/C++ | [GitHub](https://github.com/ggerganov/llama.cpp) |

## Web Development

| Name | Description | Link |
|------|-------------|------|
| next.js | React framework | [GitHub](https://github.com/vercel/next.js) |
| tailwindcss | Utility-first CSS framework | [GitHub](https://github.com/tailwindlabs/tailwindcss) |
```

## 示例 3：详细风格索引

适用场景：大量收藏（200+），需要完整信息和分类管理

```markdown
# 🌟 GitHub Stars 完整索引

> **最后更新**: 2024-02-04  
> **收藏总数**: 358 个代码库  
> **分类数量**: 12 个主分类

## 📖 索引说明

本索引整理了我在 GitHub 上收藏的所有优质代码库，按照技术领域和实际用途进行分类。每个代码库都包含详细的元数据信息，便于快速查找和评估。

### 字段说明

- **⭐ Stars**: GitHub 星标数（k=千）
- **🔱 Forks**: Fork 数量
- **🏷️ License**: 开源协议
- **📅 更新**: 最近更新时间
- **🔥 标签**: 活跃度和推荐标签
```

## 最佳实践总结

1. **选择合适的风格**: 根据收藏数量和需求选择索引风格
2. **保持一致性**: 同一文档内使用统一的表格格式
3. **适度使用图标**: 增强可读性但不要过度
4. **定期更新**: 保持信息的时效性
5. **添加说明**: 为分类添加简短说明帮助理解
6. **考虑可维护性**: 大型索引考虑分文件管理
