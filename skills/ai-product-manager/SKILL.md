# AI产品经理

## 任务目标
本 Skill 用于辅助AI产品经理在Agent产品全生命周期中进行产品调研分析、需求挖掘、PRD撰写、原型设计、能力边界定义、人机协同流程构建、效果评估体系优化以及数据驱动的迭代规划。

能力包含：
- 竞品分析与市场趋势洞察
- 用户痛点识别与需求挖掘
- 撰写结构化的产品需求文档（PRD）
- 设计可落地的产品原型
- 定义Agent的能力边界与职责范围
- 设计高效的人机协同工作流程
- 构建可量化的效果评估体系
- 规划数据收集与反馈迭代机制

触发条件：
- 规划新的AI产品或功能
- 进行产品调研和竞品分析
- 分析市场趋势和用户痛点
- 进行需求分析和用户调研
- 撰写产品需求文档（PRD）
- 设计产品原型和交互流程
- 定义Agent能力边界
- 优化人机协同流程
- 建立或优化效果评估指标
- 规划数据收集与迭代策略

## 前置准备
建议准备以下工具以提高效率：
- **市场调研**：艾瑞咨询、易观分析、QuestMobile
- **竞品分析**：竞品官网、应用商店、社交媒体
- **PRD撰写**：飞书文档、Google Docs、Notion
- **原型设计**：墨刀、Figma、Axure

具体工具使用指南见 [references/tools-guide.md](references/tools-guide.md)

## 操作步骤

### 1. 产品调研分析

**目标**：深入了解竞品和市场，识别机会和痛点，为产品决策提供依据。

**执行要点**：
1. 竞品分析：
   - 竞品选择：直接竞品、间接竞品、潜在竞品
   - 调研方法：功能对比、体验评估、用户评价分析
   - 分析维度：产品定位、核心功能、商业模式、用户体验
2. 市场趋势分析：
   - 市场规模：市场规模、增长率、细分市场
   - 发展趋势：技术趋势、用户需求变化、政策环境
   - 竞争格局：市场集中度、头部玩家、竞争态势
3. 用户痛点识别：
   - 痛点来源：用户访谈、竞品差评、社交媒体、数据分析
   - 分析方法：痛点分类、影响度评估、原因分析
   - 机会识别：痛点聚类、机会排序、差异化定位

**参考文档**：[references/product-research.md](references/product-research.md)

### 2. 需求挖掘

**目标**：深度挖掘用户真实需求，洞察业务痛点，明确产品定位和价值主张。

**执行要点**：
1. 用户场景分析：识别目标用户及其核心使用场景
   - 用户画像：角色、特征、行为模式
   - 使用场景：高频场景、痛点场景、机会场景
2. 需求收集：多渠道收集需求信息
   - 用户访谈：定性深度访谈、用户观察
   - 数据分析：用户行为数据、使用日志
3. 需求优先级排序：
   - 影响度评估：用户价值、商业价值
   - 实现难度评估：技术可行性、资源需求
   - ROI分析：价值与成本比
4. 需求输出：形成结构化需求文档
   - 用户故事
   - 需求优先级矩阵
   - MVP功能清单

**参考文档**：[references/requirement-mining.md](references/requirement-mining.md)

### 3. 撰写PRD

**目标**：产出清晰、完整、可执行的产品需求文档。

**执行要点**：
1. 文档结构：按照标准PRD框架组织内容
   - 项目背景与目标
   - 用户画像与场景
   - 功能需求描述
   - 非功能需求（性能、安全、兼容性）
   - 数据需求（数据结构、埋点）
   - 验收标准
2. 需求描述规范：
   - 使用清晰的语言描述功能
   - 包含输入输出、交互流程
   - 明确异常处理和边界条件
3. 可视化辅助：
   - 流程图（业务流程、数据流程）
   - 原型图（页面结构、交互设计）
   - 数据字典（字段定义、关联关系）

**参考文档**：[references/prd-template.md](references/prd-template.md)

### 4. 设计产品原型

**目标**：设计可落地的产品原型，验证交互设计和用户体验。

**执行要点**：
1. 信息架构设计：
   - 功能模块划分
   - 页面层级结构
   - 导航路径设计
2. 交互设计：
   - 用户操作流程
   - 交互反馈机制
   - 异常状态处理
3. 界面设计：
   - 页面布局
   - 元素样式
   - 视觉层次
4. 原型验证：
   - 可用性测试
   - 用户反馈收集
   - 迭代优化

**参考文档**：[references/prototype-design.md](references/prototype-design.md)

### 5. 定义Agent能力边界

**目标**：明确Agent能够可靠完成的任务范围，避免过度承诺和用户预期偏差。

**执行要点**：
1. 识别核心能力：基于产品定位，列举Agent应具备的核心能力（如文本理解、推理分析、创意生成等）
2. 评估技术可行性：对照当前AI技术边界，区分：
   - 可靠实现（准确率>90%）
   - 辅助实现（需要人工审核）
   - 暂不支持（技术未成熟）
3. 定义边界条件：明确能力的约束条件（如输入长度、领域范围、复杂度限制）
4. 设计拒绝机制：对于超出边界的需求，设计友好的拒绝话术和降级方案

**参考文档**：[references/agent-boundary.md](references/agent-boundary.md)

### 6. 构建人机协同流程

**目标**：设计合理的任务分工与协作机制，最大化人机协同效率。

**执行要点**：
1. 任务分类：按照以下维度分类任务
   - 自动化任务（Agent独立完成）
   - 半自动化任务（Agent草稿+人工确认）
   - 人工主导任务（Agent辅助决策）
2. 流程设计：绘制协作流程图，明确：
   - 任务启动方式
   - 人机交接节点
   - 异常处理机制
   - 反馈收集方式
3. 交互优化：设计清晰的用户引导和操作提示

**参考文档**：[references/human-ai-workflow.md](references/human-ai-workflow.md)

### 7. 优化效果评估体系

**目标**：建立多维度、可量化的评估指标体系，客观衡量Agent表现。

**执行要点**：
1. 维度设计：从以下维度设计指标
   - 准确性（如任务完成率、错误率）
   - 效率（如响应时间、处理速度）
   - 用户体验（如满意度、NPS）
   - 业务价值（如转化率、成本降低）
2. 指标定义：为每个指标定义
   - 计算方式
   - 数据来源
   - 基准值和目标值
3. 评估方法：选择合适的评估方法
   - 自动化评估（如规则匹配、对比模型）
   - 人工评估（如专家打分、A/B测试）
   - 用户反馈（如问卷、评分）

**参考文档**：[references/evaluation-system.md](references/evaluation-system.md)

### 8. 驱动数据飞轮迭代

**目标**：建立数据收集、分析、优化的闭环机制，持续提升产品表现。

**执行要点**：
1. 数据收集：设计数据收集方案
   - 关键事件埋点（如任务启动、完成、失败）
   - 用户行为追踪（如点击、编辑、反馈）
   - 质量标签标注（如正确、错误、需改进）
2. 分析机制：建立数据分析流程
   - 定期数据看板（日报/周报/月报）
   - 问题根因分析（如错误聚类、失败路径分析）
   - 机会识别（如高频需求、痛点场景）
3. 迭代规划：基于数据分析制定迭代计划
   - 优先级排序（影响度×频度）
   - 实验设计（如A/B测试、灰度发布）
   - 效果验证（如指标对比、用户反馈）

**参考文档**：[references/evaluation-system.md](references/evaluation-system.md)（迭代部分）

### 分支处理

**当产品处于0-1阶段**：
- 优先产品调研和需求挖掘，明确产品定位
- 快速设计MVP原型，验证核心假设
- 定义Agent核心能力边界
- 采用轻量级评估指标（如任务完成率、用户满意度）
- 重点收集定性反馈（如用户访谈、使用场景观察）

**当产品处于1-10阶段**：
- 优化人机协同流程，降低人工干预成本
- 完善原型设计，提升用户体验
- 构建完整的评估体系，关注业务指标（如转化率、留存率）
- 建立自动化数据分析和迭代机制

**当产品处于10-100阶段**：
- 基于数据挖掘深层需求，规划新产品功能
- 优化产品架构和性能
- 建立产品矩阵和生态体系

## 注意事项

- 产品调研要深入，不仅要看表面功能，更要理解背后的逻辑和策略
- 竞品分析要客观，既要学习优点，也要发现不足和机会
- 需求挖掘要深入理解用户痛点，避免表面需求
- PRD撰写要结构清晰、描述完整，确保开发可执行
- 原型设计要注重用户体验，验证交互逻辑
- 工具选择要结合团队协作习惯和项目需求，详见[tools-guide.md](references/tools-guide.md)
- 充分利用智能体的分析推理能力，将复杂问题拆解为可执行的步骤
- 在设计评估指标时，确保指标的可测量性和可操作性
- 人机协同流程应以用户为中心，简化操作复杂度
- 数据收集应遵循隐私保护原则，确保数据合规性
- 迭代规划应采用小步快跑的方式，避免过度设计

## 使用示例

### 示例1：智能客服产品调研与需求挖掘

**场景**：规划一个智能客服Agent产品。

**执行方式**：智能体主导，参考[product-research.md](references/product-research.md)和[requirement-mining.md](references/requirement-mining.md)

**关键要点**：
1. 产品调研：
   - 竞品分析：分析云问、智齿、网易七鱼等竞品
   - 市场趋势：智能客服市场年增长率30%，大模型驱动的智能客服成为趋势
   - 用户痛点：人工成本高、响应慢、服务质量不稳定、7x24小时服务难实现
2. 需求挖掘：
   - 目标用户：电商客服团队、C端用户
   - 核心痛点：FAQ咨询、订单查询、投诉处理、产品推荐
3. PRD撰写：
   - 产品定位：AI驱动的智能客服系统，提升客服效率和服务质量
   - 功能需求：FAQ自动回答、订单查询、投诉处理、知识库管理
   - 非功能需求：响应时间<3秒、可用性>99.5%、支持并发1000+

**输出**：一份包含竞品分析、市场趋势、用户痛点、需求分析、功能描述的完整调研报告和PRD文档

### 示例2：代码助手竞品分析与产品规划

**场景**：设计一个AI代码助手，需要进行竞品分析和产品规划。

**执行方式**：智能体主导，参考[product-research.md](references/product-research.md)和[agent-boundary.md](references/agent-boundary.md)

**关键要点**：
1. 竞品分析：
   - 直接竞品：GitHub Copilot、Cursor、CodeLlama
   - 功能对比：
     - Copilot：VSCode插件、上下文感知、多语言支持
     - Cursor：内置编辑器、自然语言编辑、长上下文
     - CodeLlama：开源模型、本地部署、成本优势
   - 竞品劣势：理解业务场景能力弱、学习成本高
2. 市场趋势：
   - 开发者工具市场持续增长
   - AI辅助编程成为标配
   - 企业级需求：私有化部署、安全可控
3. 差异化定位：
   - 更强的业务理解能力
   - 更友好的人机协同体验
   - 支持企业级私有化部署
4. 能力边界：
   - 核心能力：代码生成、代码补全、代码解释、Bug修复建议
   - 可靠实现：代码补全（准确率>95%）、代码解释
   - 辅助实现：代码生成（需人工Review）、Bug修复（需验证）

**输出**：一份包含竞品分析报告、市场趋势分析、差异化定位、能力边界的设计文档

### 示例3：内容生成平台原型设计与评估体系构建

**场景**：构建一个AI内容生成平台，需要设计原型和评估体系。

**执行方式**：智能体主导，参考[prototype-design.md](references/prototype-design.md)和[evaluation-system.md](references/evaluation-system.md)

**关键要点**：
1. 原型设计：
   - 信息架构：首页、创作页、内容管理、模板库、个人中心
   - 交互流程：用户选择类型 → 输入参数 → Agent生成 → 用户编辑 → 导出
   - 界面设计：左侧工具栏、中间编辑区、右侧预览区
2. 评估体系：
   - 准确性：事实错误率<2%、内容相关性>90%
   - 用户体验：满意度>4.2、采纳率>75%
   - 业务价值：内容生成效率提升>50%、用户留存>30%

**输出**：一份包含原型图（流程图、页面结构）和评估指标体系的设计文档
