## 工具选择建议

### 工具对比矩阵

| 工具类型 | 推荐工具 | 适用场景 | 优势 | 限制 |
|---------|---------|---------|------|------|
| **PRD撰写** | 飞书文档 | 团队协作、实时编辑 | 协作性强、模板丰富、版本管理 | 私有化部署受限 |
| | Google Docs | 全球化团队 | 多语言、协作便捷 | 国内访问慢 |
| | Notion | 知识库+文档 | 模块化、易组织 | 学习成本较高 |
| **原型设计** | 墨刀 | 快速原型 | 中文界面、易上手、组件丰富 | 高保真设计较弱 |
| | Figma | 设计协作 | 实时协作、设计规范 | 学习曲线陡峭 |
| | Axure | 复杂交互 | 交互逻辑强、功能全面 | 界面较旧、协作弱 |
| **需求管理** | 飞书多维表格 | 轻量需求管理 | 与文档集成、易用 | 复杂流程支持弱 |
| | Jira | 专业需求管理 | 工作流强、插件丰富 | 配置复杂、学习成本高 |
| | Notion表格 | 灵活管理 | 自由度高、易用 | 权限控制较弱 |
| **流程设计** | Draw.io | 流程图 | 免费、简洁 | 模板较少 |
| | XMind | 思维导图 | 思维整理清晰 | 流程图功能弱 |
| | ProcessOn | 综合图表 | 模板丰富、免费版受限 | 导出需付费 |

### 选择原则

**1. 团队协作优先**
- 团队已使用什么工具 → 优先沿用，降低学习成本
- 团队规模 → 小团队选择轻量工具，大团队选择专业工具
- 协作需求 → 考虑实时协作、版本管理、权限控制

**2. 项目需求匹配**
- 简单原型 → 墨刀
- 高保真设计 → Figma
- 复杂交互 → Axure
- 快速验证 → 手绘或线框图

**3. 学习成本考虑**
- 初学者 → 墨刀、飞书
- 有设计经验 → Figma
- 专业原型师 → Axure

## 墨刀原型设计

### 1. 墨刀简介

墨刀是一款在线原型设计工具，支持快速创建可交互的原型，适合产品经理快速验证产品想法。

**核心功能**：
- 拖拽式组件库
- 交互逻辑设计
- 预览和分享
- 团队协作

**适用场景**：
- 低保真原型（线框图）
- 中保真原型（带交互）
- 快速验证产品想法

### 2. 快速上手

#### 注册登录
1. 访问墨刀官网：https://modao.cc
2. 注册账号（支持邮箱、手机、第三方登录）
3. 选择免费版或付费版

#### 创建项目
1. 点击"新建项目"
2. 选择项目类型：
   - 移动端（iPhone、Android）
   - PC端（Web、桌面应用）
   - 自定义尺寸
3. 命名项目
4. 选择模板（可选）

### 3. 页面设计

#### 界面布局
```
┌─────────────────────────────────────┐
│ 顶部工具栏：保存、预览、分享、设置        │
├───────┬─────────────────────────────┤
│       │                             │
│ 组件   │      画布区域                  │
│ 库     │                             │
│       │                             │
├───────┴─────────────────────────────┤
│ 底部：页面列表、图层、属性              │
└─────────────────────────────────────┘
```

#### 常用组件
| 组件类别 | 常用组件 | 使用场景 |
|---------|---------|---------|
| 基础组件 | 矩形、圆形、文本、图片 | 布局框架、占位图 |
| 表单组件 | 输入框、按钮、下拉框、复选框 | 表单页面 |
| 导航组件 | 标签栏、导航栏、抽屉、弹窗 | 页面导航 |
| 列表组件 | 列表、卡片、表格 | 列表展示 |
| 媒体组件 | 音频、视频、轮播图 | 媒体展示 |

#### 设计步骤
**步骤1：创建页面**
1. 点击左下角"+"按钮
2. 选择页面类型（空白页/模板）
3. 命名页面（如"首页"、"登录页"）

**步骤2：拖拽组件**
1. 从左侧组件库选择组件
2. 拖拽到画布
3. 调整位置和大小

**步骤3：编辑组件属性**
1. 选中组件
2. 在右侧属性面板编辑：
   - 文本内容
   - 样式（颜色、字体、圆角）
   - 尺寸和位置

**步骤4：添加交互**
1. 选中需要添加交互的组件（如按钮）
2. 点击右侧"交互"面板
3. 添加交互事件：
   - 触发方式：点击、悬停、双击
   - 动作：跳转页面、打开链接、显示/隐藏
   - 动画：无/淡入/滑动/缩放
   - 时长：0.3秒（默认）

**示例：登录页面交互**
```
组件：登录按钮
交互设置：
- 触发：点击
- 动作：跳转页面
- 目标：首页
- 动画：淡入
- 时长：0.3秒
```

### 4. 交互逻辑设计

#### 页面跳转
```
设置方式：
1. 选中按钮/链接
2. 添加交互：点击 → 跳转页面
3. 选择目标页面
4. 选择动画效果（可选）
```

#### 弹窗显示
```
设置方式：
1. 选中触发按钮
2. 添加交互：点击 → 显示弹窗
3. 选择弹窗组件
4. 设置动画效果

关闭弹窗：
1. 选中弹窗内的关闭按钮
2. 添加交互：点击 → 隐藏弹窗
3. 选择要隐藏的弹窗组件
```

#### 状态切换
```
设置方式：
1. 创建多个状态（如选中、未选中）
2. 添加交互：点击 → 切换状态
3. 选择目标状态
```

#### 表单交互
```
设置方式：
1. 添加输入框组件
2. 添加交互：输入时 → 显示/隐藏提示
3. 添加提交按钮交互：点击 → 验证 → 跳转
```

### 5. 预览与分享

#### 预览原型
1. 点击顶部"预览"按钮
2. 选择预览模式：
   - 桌面预览
   - 手机预览
3. 测试交互效果

#### 分享原型
1. 点击"分享"按钮
2. 选择分享方式：
   - 生成链接：复制链接分享给团队
   - 嵌入代码：嵌入到其他平台
3. 设置权限：
   - 查看权限（公开/密码/仅限指定人）
   - 评论权限（允许/禁止）
   - 下载权限（允许/禁止）

### 6. 协作功能

#### 邀请协作
1. 点击右上角"邀请"
2. 输入邮箱或用户名
3. 设置权限：
   - 查看者：只能查看
   - 编辑者：可编辑原型
   - 管理员：完全控制

#### 评论反馈
1. 在预览模式下选中组件
2. 点击"评论"按钮
3. 输入评论内容
4. @相关人员进行讨论

### 7. 最佳实践

**设计规范**：
- 保持设计一致性（颜色、字体、间距）
- 使用组件库提高效率
- 合理使用母版（Header、Footer等公共部分）

**交互设计**：
- 交互要简单直观
- 避免过度复杂的动画
- 提供明确的用户反馈

**文件管理**：
- 合理命名页面和组件
- 使用文件夹组织页面
- 定期备份

### 8. 常见问题

**Q：如何导出原型？**
A：点击"文件" → "导出"，支持导出为PDF、HTML、图片等格式。

**Q：如何使用母版？**
A：在左侧面板选择"母版"，创建母版后可在多个页面复用。

**Q：如何制作动态效果？**
A：使用"状态"功能创建不同状态的组件，通过交互切换状态实现动态效果。

## 飞书文档PRD撰写

### 1. 飞书文档简介

飞书文档是飞书套件中的在线文档工具，支持多人实时协作、模板管理、版本控制，适合团队协作撰写PRD。

**核心功能**：
- 实时多人协作
- 丰富的排版工具
- 模板库
- 版本历史
- 评论和@提醒

**适用场景**：
- 团队协作撰写PRD
- 需要频繁更新和维护的文档
- 需要多人评审的文档

### 2. 创建PRD文档

#### 创建方式
**方式1：使用模板**
1. 打开飞书文档
2. 点击"新建"
3. 选择"模板"
4. 搜索"PRD"或"产品需求"
5. 选择合适的模板创建

**方式2：从空白开始**
1. 打开飞书文档
2. 点击"新建文档"
3. 选择"空白文档"
4. 按照PRD模板结构撰写

#### 文档结构建议
```
文档标题：[产品名称] PRD
文档标签：#PRD #产品名称 #版本号

一、项目背景
二、用户画像
三、产品概述
四、功能需求
   4.1 功能1
   4.2 功能2
五、非功能需求
六、数据需求
七、AI能力需求
八、验收标准
```

### 3. 排版技巧

#### 标题层级
```
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（小节）
```

**快捷键**：
- `Ctrl/Cmd + 1`：一级标题
- `Ctrl/Cmd + 2`：二级标题
- `Ctrl/Cmd + 3`：三级标题

#### 列表
```
无序列表：
- 列表项1
- 列表项2
  - 子项1
  - 子项2

有序列表：
1. 列表项1
2. 列表项2
   1. 子项1
   2. 子项2
```

#### 表格
```
创建表格：输入 "|" 或点击表格图标
| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| user_id | String | 是 | 用户ID |
```

#### 代码块
```
```python
def hello():
    print("Hello, World!")
```
```

#### 引用
```
> 重要提示：...
```

#### 分隔线
```
---
```

### 4. 协作功能

#### 邀请协作
1. 点击右上角"分享"
2. 输入邮箱或选择飞书用户
3. 设置权限：
   - 可查看
   - 可评论
   - 可编辑

#### 评论功能
```
使用场景：

操作步骤：

使用场景：

4. 可以对比不同版本

#### 恢复版本
1. 在历史版本中选择要恢复的版本

#### 版本命名

```

### 7. 最佳实践

### 8. 常见问题

## 飞书多维表格需求管理

### 1. 飞书多维表格简介

飞书多维表格是一款轻量级的表格工具，支持字段类型自定义、视图切换、协作编辑，适合需求管理和追踪。

**核心功能**：
- 多种字段类型（文本、数字、选项、日期、人员等）
- 多视图切换（表格、看板、甘特图、日历）
- 数据关联和引用
- 协作编辑
- 数据筛选和排序

**适用场景**：
- 需求池管理
- Bug追踪
- 任务管理
- 数据收集

### 2. 创建需求表格

#### 创建方式
1. 打开飞书多维表格
2. 点击"新建表格"
3. 选择模板（可选）
4. 命名表格（如"需求池"）

#### 字段设计
```
基础字段：
- 需求ID（自动编号）
- 需求标题（文本）
- 需求描述（多行文本）
- 需求类型（选项：功能/优化/bug/体验）
- 优先级（选项：P0/P1/P2/P3）
- 状态（选项：待评审/开发中/测试中/已完成/已发布）
- 提出人（人员）
- 提出时间（日期时间）
- 期望完成时间（日期）
- 模块（选项）
- 复杂度（选项：高/中/低）
- 预估工时（数字）
- 实际工时（数字）
- 附件（文件）
- 关联PRD（关联字段）
```

#### 创建字段步骤
1. 点击表格标题行右侧"+"
2. 选择字段类型
3. 命名字段
4. 设置字段属性（如选项列表）
5. 确认创建

### 3. 视图管理

#### 表格视图
**适用场景**：查看所有需求的详细信息
**功能**：
- 列筛选：选择显示哪些字段
- 行筛选：按条件筛选需求
- 排序：按字段排序
- 分组：按字段分组显示

#### 看板视图
**适用场景**：按状态或优先级管理需求
**设置方式**：
1. 点击"新建视图"
2. 选择"看板"
3. 选择分组字段（如"状态"）
4. 配置看板列

**示例**：
```
待评审 | 开发中 | 测试中 | 已完成 | 已发布
─────────────────────────────────────
需求1  | 需求3  | 需求5  | 需求7  | 需求9
需求2  | 需求4  | 需求6  | 需求8  |
```

#### 甘特图视图
**适用场景**：查看需求的时间规划和进度
**设置方式**：
1. 点击"新建视图"
2. 选择"甘特图"
3. 选择开始时间字段（如"提出时间"）
4. 选择结束时间字段（如"期望完成时间"）

#### 日历视图
**适用场景**：按日期查看需求安排
**设置方式**：
1. 点击"新建视图"
2. 选择"日历"
3. 选择日期字段（如"期望完成时间"）

### 4. 需求管理流程

#### 需求录入
```
步骤1：点击"新建记录"
步骤2：填写需求信息
  - 需求标题
  - 需求描述
  - 需求类型
  - 提出人
步骤3：设置初始状态（默认"待评审"）
步骤4：保存
```

#### 需求评审
```
操作步骤：
1. 筛选"待评审"状态的需求
2. 逐个评审需求
3. 设置优先级（P0/P1/P2/P3）
4. 更新状态（开发中/已拒绝/已暂缓）
5. 添加评审备注（在"需求描述"或"评论"中）
```

#### 需求追踪
```
定期检查：
1. 筛选"开发中"的需求，确认进度
2. 筛选"测试中"的需求，跟进测试结果
3. 筛选"已完成"的需求，确认发布时间
4. 更新需求状态和实际工时
```

#### 需求统计
```
统计维度：
- 需求数量统计（按类型、状态、模块）
- 工时统计（预估vs实际）
- 完成率统计
- 周期时间统计

使用方式：
1. 使用表格视图的筛选和分组功能
2. 使用图表功能（如仪表盘插件）
3. 导出数据到Excel进行高级分析
```

```

#### 关联文档
```
使用场景：关联需求对应的PRD、原型等文档

操作步骤：
1. 创建关联字段
2. 选择关联的文档类型（如飞书文档）
3. 在需求记录中关联对应文档
4. 点击可快速跳转查看
```

### 6. 最佳实践

**字段命名规范**：
- 使用统一、清晰的字段名
- 避免缩写和模糊的表述
- 添加字段说明

**状态流转规范**：
```
标准流程：
待评审 → 开发中 → 测试中 → 已完成 → 已发布

分支流程：
待评审 → 已拒绝（不满足需求）
待评审 → 已暂缓（优先级低）
开发中 → 待评审（需求变更）
测试中 → 开发中（测试不通过）
```

**优先级定义**：
```
P0：必须有，阻碍上线
P1：应该有，重要但不紧急
P2：可以有，锦上添花
P3：暂不做，暂不考虑
```

**定期维护**：
- 每周评审新需求
- 每月清理过期需求
- 每季度统计需求完成情况

#### 界面布局
```

#### 常用工具
| 工具 | 快捷键 | 功能 |
|-----|-------|------|
| 选择工具 | V | 选择、移动、缩放 |
| 框架工具 | F | 创建框架（类似画板） |
| 矩形工具 | R | 绘制矩形 |
| 钢笔工具 | P | 绘制矢量图形 |

| 手形工具 | H | 平移画布 |
| 缩放工具 | Z | 缩放画布 |

#### 设计步骤
**步骤1：创建框架**
1. 按F键或点击框架工具
2. 选择预设尺寸（如iPhone、Web）
3. 在画布上绘制框架

**步骤2：绘制组件**
1. 选择绘制工具（R绘制矩形，T添加文本）
2. 在框架内绘制
3. 调整位置和大小

**步骤3：设置样式**
1. 选中组件
2. 在右侧属性面板设置：
   - 填充（Fill）：颜色、渐变、图片
   - 描边（Stroke）：边框样式
   - 效果（Effects）：阴影、模糊
   - 文本：字体、字号、字重、行高

**步骤4：使用自动布局**
1. 选中多个组件
2. 右键选择"Add auto layout"（Shift+A）
3. 设置布局方向（横/纵）
4. 调整间距和对齐方式

### 4. 组件系统

#### 创建组件
1. 选中要复用的设计元素
2. 右键选择"Create component"（Ctrl/Cmd+Alt+K）
3. 组件会显示紫色图标

#### 使用组件
1. 拖拽组件到画布
2. 组件会自动创建实例（Instance）

#### 编辑组件
1. 双击组件实例
2. 修改后可以应用到所有实例
3. 或"Detach instance"独立编辑

#### 组件属性（Variants）
创建组件的不同状态：
1. 选中组件
2. 点击属性面板"Variants"
3. 添加属性（如按钮状态：默认、悬停、点击）
4. 创建不同状态的变体

### 5. 原型设计

#### 创建原型
1. 选中要添加交互的元素
2. 点击右侧"Prototype"面板
3. 拖拽连接线到目标页面
4. 设置交互属性：
   - Trigger（触发）：点击、悬停、拖拽
   - Action（动作）：导航到、打开、滚动
   - Animation（动画）：溶解、滑动、弹跳

#### 预览原型
1. 点击右上角"Play"按钮
2. 或按快捷键 `Ctrl/Cmd + P`
3. 测试交互效果

#### 分享原型
1. 点击右上角"Share"按钮
2. 复制分享链接
3. 设置权限（可查看、可评论）
4. 发送给团队

### 6. 协作功能

3. 设置权限：
   - Can view（仅查看）
